- Dialogue speech uses `─` (U+2500) as the sole separator between speech and narration. The em-dash (`—`) is forbidden.
- Thoughts, titles, and inscriptions use guillemets `«...»`.
- No quotation marks around spoken speech.
</rule>

<output_format>
Return your corrections as a JSON array of diff objects. Do NOT return the entire corrected text.

Each diff object must have this structure:
[
  {
    "chunk_index": 0,
    "find": "exact string to be replaced",
    "replace": "the new string that will replace it"
  }
]

Rules for diffs:
1. The `find` string MUST appear EXACTLY ONCE in the `content_target` of the specified `chunk_index`. Zero or multiple matches → diff is rejected.
2. Make `find` long enough to be unique but concise (usually a full sentence or phrase).
3. No leading or trailing whitespace in `find` unless required for uniqueness.
4. If no corrections needed, return an empty array: []
5. Output ONLY the JSON array. No markdown formatting or explanations.
6. CRITICAL — CHARACTER FIDELITY: Copy `find` character-for-character from content_target. Do NOT substitute or normalise punctuation: hyphen-minus U+002D `-`, en-dash U+2013 `–`, em-dash U+2014 `—`, and box-drawing dash U+2500 `─` are four distinct characters. If the source text contains `-`, your `find` must also contain `-`, not `–` or `—`. Any character substitution will cause the diff to be silently rejected.

Example:
[
  {
    "chunk_index": 0,
    "find": "— Привет, — сказала она.",
    "replace": "─ Привет, ─ сказала она."
  },
  {
    "chunk_index": 1,
    "find": "штормовая",
    "replace": "бурная"
  }
]
</output_format>
