# Планируемые улучшения

## Приоритет 1 — Быстрые победы

- [ ] **Удалить `requirements.txt`** — использовать только `pyproject.toml` (`pip install -e .`). Устраняет рассинхронизацию зависимостей и отсутствие `rich`.
- [ ] **Вынести хардкод-параметры в `book-translator.toml`** — таймауты воркеров (120с), таймаут вычитки (300с), retry (3 попытки, 4–10с). RPS остаётся константой = 2.
- [ ] **Утилита `parse_llm_json(raw: str) -> Any`** — единый парсер JSON из ответов LLM (снятие code fences, обработка обёрток `gemini-cli`). Устраняет дублирование в `orchestrator.py` и `term_collector.py`.
- [ ] **Заменить файловые чекпоинты на поле `pipeline_stage` в `state.db`** — единый источник правды вместо `.stage_*_complete` файлов.

## Приоритет 2 — Архитектурные улучшения

- [ ] **Разбить `orchestrator.py`** на: `worker.py` (запуск gemini-cli), `pipeline.py` (конвейер), `assembler.py` (сборка финального файла).
- [ ] **Заменить `input()` на CLI-флаги** (`--docx` / `--no-docx`) — убрать интерактивный `input()` из бизнес-логики.
- [ ] **Валидация конфигурации** — проверка допустимых значений языков, диапазонов числовых параметров при загрузке `book-translator.toml`.
- [ ] **Консистентный язык промптов** — привести `default_prompts.py` к единому языку. Исследовать, какой язык Gemini воспринимает лучше для инструкций.
- [ ] **Добавить `--stage` флаг** для перезапуска конкретного этапа (`--stage proofreading`). Полезно при итерациях над промптами.

## Приоритет 3 — Развитие функционала

- [ ] **Пакетный перевод** — `book-translator translate volume-01/source/` (все файлы в директории) и `book-translator translate-all` (все тома серии).
- [ ] **Детальный `status`** — прогресс по этапам (discovery / translation / proofreading), количество обработанных чанков на каждом этапе, отображение ошибок.
- [ ] **Dry-run режим** (`--dry-run`) — показать план действий без вызовов API.
- [ ] **Экспорт в EPUB** помимо DOCX — удобство для чтения на электронных книгах.
- [ ] **Интерактивный diff-просмотр** правок вычитки в TUI перед финальной сборкой.

## Приоритет 4 — Развитие TUI

- [ ] **Полноценный интерактивный TUI-интерфейс** — выбор серии, тома, главы, параметров перевода через красивый терминальный UI. Детальное обсуждение дизайна и функциональности — отдельным этапом.
